# Aut_Sci_PPT 核心排版与行为准则 (Execution Rules)

以下 13 条铁律源自原 Agent 的长期迭代总结（memory/learnings.md），必须严格执行。

## 1. 结构与大纲
- **规则 R01**：章节标题必须采用 `1. 标题` 格式，子项必须采用 `- 要点` 格式。
- **规则 R02**：🚫 禁止使用 Markdown `##` 语法作为章节标题（TextParser 不识别）。
- **规则 R03**：精准识别论文的核心创新点 (Innovation)，并在 PPT 中以红色高亮突出。

## 2. 图像与媒体
- **规则 R04**：优先调用 `Sh_Sci_Fig` 技能提取 600 DPI 高清图片。
- **规则 R05**：🚫 禁止从 PDF 二进制流中直接抠取低质量位图图片。
- **规则 R06**：图像最小宽度必须 >= 300px，否则会因显示不清被判定为错误。

## 3. 公式规范
- **规则 R07**：所有 LaTeX 公式必须异步渲染为高清 PNG 图片嵌入。
- **规则 R08**：公式图片必须保持 transparency（透明背景），禁止带白底干扰背景色。

## 4. 行为透明度
- **规则 R09**：操作预计超过 10 秒（如解析 PDF），必须第一时间告知用户预计时长。
- **规则 R10**：🚫 禁止生成中间 Markdown 文本让用户处理，必须直接输出最终可用的 `.pptx`。

## 5. 样式执行
- **规则 R11**：色值必须严格匹配 `#1E3A5F` (主色) 和 `#EE0000` (标红)。
- **规则 R12**：排版禁止出现文字溢出或图文重叠。

## 6. 交互原则
- **规则 R13**：回复应简明专业，一句能说明白的事情不用两句，跳过所有 AI 礼貌填充语。
